## 三、将半导体划分为高/中/低端：与数据和政策的对应

### 3.1 概念上如何划分

结合 BIS 门槛、CRS/CSIS 描述，可以这样定义：

1. **高端芯片（High-end）**

   * 满足 BIS 规则中 3A090/4A090 所规定的**计算性能 + 带宽 + 能效**门槛（用于 AI 训练、超级计算等）；
   * 包括：高端 GPU/AI 加速器、高性能 CPU/FPGA、HBM、先进 DRAM 等；
   * 高度集中在少数企业（Nvidia、AMD、Intel 等），对军事、情报、核试验模拟极为关键。

2. **中端芯片（Mid-end）**

   * 技术上落后于高端门槛，但仍用于服务器、PC、网络设备等一般计算场景；
   * — 通常在 10–28nm 或等效水平，虽不被列为“最敏感”，但对数字经济仍很重要。

3. **低端（成熟制程）芯片（Low-end / Legacy）**

   * 包含模拟芯片、功率器件、微控制器、车规芯片、家电芯片等，工艺节点 ≥28nm；
   * CSIS 指出，**legacy 芯片在经济与国防供应链中同样关键**，并且中国正在快速扩张这部分产能。

### 3.2 与数据的映射（可操作口径）

在实证上，我们无法直接观察“工艺节点”，但可以用海关编码 + 外部信息做一个可接受的近似：

1. **按 HS4/HS6 口径：**

   * HS4=8542：全部集成电路；
   * HS6 子项中：

     * 854231：Processors and controllers → 可以作为**高端逻辑芯片 proxy**；
     * 854232：Memories → 进一步按外部资料估算高端 HBM / DRAM 所占比例（可以设一个权重 (\theta_{HBM})）；
     * 854239：Others → 大部分视为中端或低端，根据后续假设拆分。

2. **在模型中设三个子市场的规模：**

   * 以 2022 或 2023 为基年，令美国出口/进口 8542 总额为 (Trade^{US}*{8542})，再用一组权重 ((\alpha_H, \alpha_M, \alpha_L)) 拆分为：
     [
     Trade^{US}*{H} = \alpha_H \cdot Trade^{US}*{8542}, \quad
     Trade^{US}*{M} = \alpha_M \cdot Trade^{US}*{8542}, \quad
     Trade^{US}*{L} = \alpha_L \cdot Trade^{US}_{8542},\quad \alpha_H+\alpha_M+\alpha_L=1
     ]
   * 权重可以参考：

     * SIA Factbook 中按产品类别（逻辑、存储、模拟、离散等）的销量占比；
     * 文献中“用于 AI / 高性能计算的芯片占全球市场约 10–20%”这一类估计（可在论文中作为**合理假设**说明）。

3. **地区层面：**

   * 使用 `TradeData_11_21...` 提供的**美国与中国、欧盟、日韩等的 8542 贸易流**，分别标定 US–CN 与 US–ROW 在高/中/低端上的份额；
   * 再结合 `exports_CN_sector.csv` 中电气设备/精密仪器等对华出口变化，作为出口管制冲击前后的参考。

## 四、核心数学模型框架

整体采用**“分段-分区-分期”的部分均衡 + 动态技术模块**。

### 4.1 市场结构与变量

记：

* 区域 (i \in {US, CN, O})：美国、中国、其他国家；
* 芯片类型 (s \in {H, M, L})：高、中、低端；
* 年份 (t \in {2020,\dots,2030})（可以取 2023 为基准，向前/向后模拟）。

主要变量：

* (Q_{i,s,t}^{prod})：区域 (i) 在类型 (s) 的产量；
* (Q_{i\to j,s,t})：从 (i) 出口至 (j) 的类型 (s) 数量（或价值）；
* (P_{i,s,t})：区域 (i) 生产的类型 (s) 芯片的出厂价格；
* (\tau_{j,s,t}^i)：区域 (j) 对从 (i) 进口的类型 (s) 芯片的从价关税率；
* (Sub_{i,s,t})：区域 (i) 对类型 (s) 生产的单位补贴（CHIPS 补贴）；
* (T_{i,s,t})：区域 (i) 在类型 (s) 芯片上的技术水平指标（如“等效制程节点/性能指数”）；
* (RD_{i,s,t})：区域 (i) 在类型 (s) 芯片的 R&D 支出。

### 4.2 需求：Armington 型进口需求

对区域 (j)、类型 (s) 的芯片，假定存在**原产地区分的 Armington 需求**：

* 合成品消费量：
  [
  Q^{cons}*{j,s,t} = \left[\sum*{i} \beta_{i,j,s}^{1/\sigma_s} \left(Q_{i\to j,s,t}\right)^{(\sigma_s-1)/\sigma_s}\right]^{\sigma_s/(\sigma_s-1)}
  ]
* 其中 (\sigma_s) 是替代弹性，(\beta_{i,j,s}) 基于基年贸易份额（用 `TradeData` + `trade_export_panel` 标定）。

对该合成品的**总需求**采用恒弹性需求：

[
Q^{cons}*{j,s,t} = A*{j,s,t} \left(P^{cons}*{j,s,t}\right)^{-\varepsilon*{j,s}}
]
其中 (P^{cons}_{j,s,t}) 是 CES 聚合价格指数，由各源国含税价决定：

[
P^{i\to j}*{s,t} = P*{i,s,t} (1 + \tau_{j,s,t}^i)
]

[
P^{cons}*{j,s,t} = \left[\sum_i \beta*{i,j,s} \left(P^{i\to j}_{s,t}\right)^{1-\sigma_s}\right]^{1/(1-\sigma_s)}
]

**出口管制**可以通过两种方式进入：

1. 对 US→CN 的高端芯片施加巨大的“虚拟关税”(\tau_{CN,H,t}^{US} \to \infty)（等价于禁止出口）；
2. 更精细的做法：对 US→CN 高端出口设置数量上限：
   [
   Q_{US\to CN,H,t} \leq \bar{Q}_{US\to CN,H}
   ]
   其中 (\bar{Q}) 由 2020–2022 的实际出口额乘以一个“许可比例”确定，或直接设为 0（完全管制）。这与 BIS 针对中国高端芯片基本禁运相符。

### 4.3 供给：线性/对数线性供给函数 + 成本结构

对区域 (i)、类型 (s)，可以假设简化的供给函数：

[
Q_{i,s,t}^{prod} = \gamma_{i,s,t} \left(P_{i,s,t} + Sub_{i,s,t}\right)^{\eta_{i,s}}
]

其中：

* (\eta_{i,s}>0) 为供给弹性；
* (\gamma_{i,s,t}) 包含生产率、资本存量等，可由 IPG 指数 + SIA 产量数据标定：

  * 例如令 (Q_{US,s,2023}^{prod} \propto IPG_{2023}^{US} \times \theta_s);
  * 其他地区则用全球份额减去美国份额。

**关税不会直接影响本国产量函数**，但通过拉升国内含税价，提高本土企业面对的市场价格，从而推高产量——这个反馈通过均衡条件体现。

### 4.4 贸易平衡与市场均衡条件

对每个 (i,s,t) 有：

1. **资源约束**：
   [
   Q_{i,s,t}^{prod} = \sum_j Q_{i\to j,s,t}
   ]
2. **各区域的合成品消费由需求决定**：
   [
   Q^{cons}*{j,s,t} = \sum_i Q*{i\to j,s,t}
   ]
3. **价格均衡**：对每个 (i,s,t)，出厂价 (P_{i,s,t}) 由**边际成本 + 补贴**决定，并在均衡时使供给与需求匹配；可以采用数值方法（例如用非线性求解器或迭代）求出一组 (P_{i,s,t}, Q_{i\to j,s,t})。

在实际求解时，为减小复杂度，可以：

* 把价值（美元）直接当作“数量”，因为我们关心的是贸易结构与收入；
* 把价格归一化（例如基年 2023 年各类芯片价格设为 1），只在模拟中看相对变化。

## 五、技术进步与国家安全模块

### 5.1 R&D 驱动的技术水平演化方程

根据 SIA Factbook，美国半导体企业 R&D 长期维持在销售收入的 15–20%，2023 年约为 19.5%，且 R&D 投入与技术领先高度相关。

我们可以设：

[
RD_{i,s,t} = \rho_{i,s} \cdot Sales_{i,s,t}, \quad Sales_{i,s,t} \approx P_{i,s,t} \cdot Q_{i,s,t}^{prod}
]

其中：

* 对美国，高端/中端 R&D 强度可以设为 20%，低端为 10–15%，参考 SIA；
* 对中国，根据 CSIS 报告中对大基金等政府投资规模的描述，可假设 R&D 强度略低但在提高（例如高端 15%、中端 10%、低端 8%）。

然后，用一个简单的技术进步方程：

[
T_{i,s,t+1} = T_{i,s,t} \left[1 + \phi_s \cdot \frac{RD_{i,s,t}}{Sales_{world,s,t}}\right]
]

其中 (\phi_s) 是把相对 R&D 转换为**技术进步速度**的系数（高端 (\phi_H) 最大）。

**出口管制与关税对技术的作用：**

* 出口管制减少美国对华高端芯片销售 → 降低 (Sales_{US,H,t}) → 降低 (RD_{US,H,t}) → 对 (T_{US,H,t}) 有负向影响；
* 但同时也压制了中国获取高端设备和芯片 → 降低 (T_{CN,H,t}) 的增长速度（尤其是通过禁止美国设备与服务进入中国先进晶圆厂）；
* 关税如果主要落在**从中国进口的中/低端芯片**，会促使这些产能向美国和“友岸”转移，可能增加美国低端产量和 R&D 基数。

在模型中可以用：

* 对中国 (T_{CN,H,t+1}) 减少一个额外的“管制惩罚项”；
* 对美国 (T_{US,H,t+1}) 减少一个“收入损失惩罚项”，都由 CRS/CSIS/GAO 的定性描述支持。

### 5.2 国家安全指标的构造

考虑两个维度：**供应安全**和**技术领先度**。

1. **供应安全（Supply Security）**

   对类型 (s)，美国在 t 年的供应安全度定义为：

   [
   SAF_{s,t} = 1 - \frac{Import^{risk}*{US,s,t}}{Cons*{US,s,t}}
   ]

   其中：

   * (Import^{risk}*{US,s,t})：来自“高风险国家”的进口（这里主要是中国）；可由模型中的 (Q*{CN\to US,s,t}) 计算；
   * (Cons_{US,s,t} = Q^{cons}_{US,s,t}) 总消费（用于国防和民用的综合）。

   (SAF_{s,t}) 越高说明越安全，即对“潜在对手”的依赖越低。CSIS 报告强调 legacy 芯片对汽车、军用设备的重要性，因此高/中/低三类都要考虑。

2. **技术领先度（Tech Gap）**

   对高端芯片，定义美国相对于中国的技术领先度：

   [
   Gap_{H,t} = \ln\left(\frac{T_{US,H,t}}{T_{CN,H,t}}\right)
   ]

   GAO 报告指出美国政策目标已经从“领先两代”升级为“保持尽可能大的领先”，这可以视作希望 (Gap_{H,t}) 尽量大。

3. **综合国家安全指数（NSI）**

   把上述指标加权合成：

   [
   NSI_t = \sum_{s\in{H,M,L}} w_s SAF_{s,t} + \mu \cdot Gap_{H,t}
   ]

   * 权重 (w_s) 体现不同芯片类型的重要性，通常 (w_H > w_M > w_L)；
   * (\mu>0) 用来调整技术领先在整体国家安全中的权重。

   在论文中，你可以根据文献或专家判断给出一组基准权重，并做敏感性分析。

## 六、经济效率指标：福利与财政

为了衡量“关税 vs 补贴”的效率，需要构造经济福利指标：

1. **消费者剩余（CS）**
   对每个区域 j 和类型 s，根据需求曲线计算：

   [
   CS_{j,s,t} = \int_{P^{cons}*{j,s,t}}^{\infty} Q^{cons}*{j,s,t}(p),dp
   ]

   如果使用恒弹性需求，可推导显式表达式（或者用数值积分）。

2. **生产者剩余（PS）**
   对每个区域 i、类型 s：

   [
   PS_{i,s,t} = P_{i,s,t} \cdot Q_{i,s,t}^{prod} - C_{i,s,t}(Q_{i,s,t}^{prod})
   ]

   其中成本函数 C 可简单设为二次函数或线性函数（参数用基年价格–数量校准）。

3. **政府财政：关税收入（TR）与补贴成本（SUB）**

   [
   TR_{t} = \sum_{j,s,i\neq j} \tau_{j,s,t}^i \cdot P_{i,s,t} \cdot Q_{i\to j,s,t}
   ]

   [
   SUB_{t} = \sum_{i,s} Sub_{i,s,t} \cdot Q_{i,s,t}^{prod}
   ]

4. **经济效率指标（Welfare）**

   [
   W_t = \sum_{j,s} CS_{j,s,t} + \sum_{i,s} PS_{i,s,t} + TR_t - SUB_t
   ]

   这就是一个年度总福利指标。

5. **综合目标函数**

   若希望给出“最优关税/补贴组合”，可以构造一个多目标加权函数：

   [
   Obj = \sum_t \delta^t \left[ W_t + \lambda \cdot NSI_t \right]
   ]

   其中 (\delta) 为贴现因子，(\lambda) 是国家安全相对权重。

## 七、政策情景设计与求解流程

### 7.1 基准情景（Scenario 0：2023 政策环境）

* 关税：

  * HS 8542 芯片 MFN = 0（`tariff_hs4_panel` 给出）；
  * 其他电气设备按照实际 MFN 和 Section 301 额外关税（可以用 `tariff_hs2_panel` 及 `has_additional_duty` 近似）。
* 补贴：

  * CHIPS 与科学法案开始发放投资补贴和 R&D 支持，对美国高/中端芯片给出 (Sub_{US,H}, Sub_{US,M})；
* 出口管制：

  * 按 CRS/GAO 描述，将 2022/10/7 及 2023、2024 更新的高级计算规则作为既定事实，对 US→CN 高端芯片实施数量上限或禁止；
* 时间：2020–2024 用作**校准**，2025–2029 模拟。

### 7.2 政策情景（可选组合）

你可以设计 3–4 个主要情景：

1. **Scenario 1：仅补贴（Tariff Free）**

   * 芯片继续保持 MFN=0；
   * CHIPS 补贴按计划实施；
   * 出口管制维持拜登政府末期水平（不考虑特朗普第二任期政策变化）。

2. **Scenario 2：互惠关税替代补贴（Tariff Only）**

   * 假设特朗普政府认为关税比补贴更有效，对大部分补贴项目砍掉：

     * (Sub_{US,s,t}=0)；
   * 将 HS 8542 也纳入 10% 基线关税：

     * (\tau_{US,H,t}^{CN} = \tau_{US,M,t}^{CN} = \tau_{US,L,t}^{CN} = 0.3)（假设对中国加到 30%，其他国家 10%；具体数值可据题意设定）；
   * 出口管制维持或略有放松（如允许 Nvidia H20 等特殊许可），即对 US→CN 高端出口上限适度放宽。

3. **Scenario 3：关税+补贴并用（Tariff + Subsidy）**

   * 在 Scenario 1 的补贴基础上，加上**适度**关税（如对中国 15%，对盟友 0–5%），并假设**关税收入部分用于继续财政补贴**：
     [
     SUB_t = SUB_t^{CHIPS} - \kappa \cdot TR_t
     ]
   * 分析“以税养补”的可持续性与效果。

4. **Scenario 4：针对不同芯片分级的差别关税**

   * 高端芯片继续 MFN=0（保持与盟友的供应链、避免损伤美国高端设计公司）；
   * 对中低端芯片从中国进口征收较高关税，引导汽车/家电等关键行业的**本土或友岸生产**，增强 legacy 供应安全（呼应 CSIS 关于 legacy 芯片风险的分析）。

### 7.3 具体求解步骤（你可以直接写在论文“方法”章节）

1. **数据预处理**

   * 用 `tariff_hs4_panel` 提取 8542 行，构造 2015–2024 年 MFN 基线；
   * 用两份 DataWeb Excel 将 854231（处理器）2018–2022 进出口额提出来，配合 8542 总额估算高/中/低端权重；
   * 用 `IPG3344S` 计算按年平均的产出指数，归一化为 2023=100；
   * 用 `exports_CN_sector.csv`、`trade_export_panel` 估算美国对中国、对世界在电气设备/半导体上的贸易份额；
   * 用 `duty_by_sector_year.csv` 和 `duty_total_year.csv` 校验关税收入与 MFN 水平行是否匹配。

2. **参数标定**

   * 使用基年（例如 2023）贸易额、产量指数标定 Armington 权重 (\beta_{i,j,s})；
   * 取自文献或假设的需求弹性 (\varepsilon_{j,s})、替代弹性 (\sigma_s)、供给弹性 (\eta_{i,s})；
   * 使用 SIA R&D 占比和 CRS/CSIS 信息设定 (\rho_{i,s}, \phi_s) 等技术参数。

3. **校准基准情景**

   * 令 t=2023，模拟无关税（芯片 MFN=0）+ 既定出口管制 + CHIPS 补贴的均衡；
   * 调整供给函数中的 (\gamma_{i,s,t}) 和需求参数 (A_{j,s,t})，使模型输出的贸易/产出与实际数据尽量接近（最小二乘标定）。

4. **前向模拟与情景对比**

   * 对每个 t 和每个政策情景：

     1. 设定关税 (\tau_{j,s,t}^i)、补贴 (Sub_{i,s,t})、出口管制上限 (\bar{Q}_{US\to CN,H})；
     2. 求解静态均衡，得到 (P_{i,s,t}, Q_{i,s,t}^{prod}, Q_{i\to j,s,t})；
     3. 根据销售额计算 (RD_{i,s,t})，更新技术 (T_{i,s,t+1})；
     4. 计算福利 (W_t) 和国家安全指数 (NSI_t)；
   * 累积得到 2025–2029 的时间路径 (W_t, NSI_t)。

5. **结果展示与分析**

   * 绘制：

     * 美国高/中/低端芯片产量、出口、进口随时间变化；
     * 美国对中国 vs 对其他国家的进口依赖度变化；
     * 高端技术差距 Gap_H_t 和三类芯片供应安全 SAF_s_t 的变化；
     * (W_t) 与 (NSI_t) 的权衡曲线（可以画在同一图中，查看某情景是否实现“以小的福利损失换来显著安全提升”，或反之）。

6. **敏感性分析**

   * 对关键参数（需求弹性、供给弹性、R&D 效率 (\phi_s)、权重 (\lambda,w_s,\mu)）做敏感性分析；
   * 检查模型结论是否稳健：例如，“对中国中低端芯片征关税 + 保持对高端芯片零关税 + 维持 CHIPS 补贴”是否在多数参数设置下都表现为：

     * 适度增加美国本土制造和供应安全；
     * 同时对技术领先和整体福利的损害相对较小。

## 八、你可以在论文中重点强调的结论方向（从模型出发）

具体数值要等你跑完模拟才知道，但模型本身能够支撑你提出以下类型的结论（供你参考）：

1. **如果对所有芯片一刀切征收 10%+ 的关税**：

   * 美国高端芯片进口减少，部分产量转移回美国或友岸，但由于高端芯片本身 MFN 已为 0，关税会明显抬高下游 AI/云计算/国防系统成本；
   * 同时，对盟友（台、韩、荷等）的损害可能弱化联盟协同，而这在出口管制体系中是关键；
   * 结合 R&D–技术模块，很可能发现：**过高的普遍关税会在长期削弱美国高端芯片企业的全球销售与 R&D 投入，从而反而缩小技术差距**。

2. **若采用“高端零关税、中低端差别关税 + 补贴”的组合**：

   * 可以显著降低美国在 legacy 芯片上对中国的依赖，提高 SAF_L, SAF_M；
   * 同时维持高端领域与盟友间的自由流动，保持美国设计企业在全球高端市场的销售与 R&D 基数；
   * 用关税收入部分支持 CHIPS 计算型制造补贴，可以在税收和财政之间形成相对可持续的平衡。

3. **出口管制和关税之间存在显著的交互效应**：

   * CRS 与 GAO 的材料表明：出口管制已经在压制中国高端能力，同时企业绕道和降级设计的行为存在；
   * 模型可以展示：**单纯进一步提升关税，对已经被管制的高端芯片边际效果有限，而对中低端的关税设置则更敏感地影响供应安全**。
