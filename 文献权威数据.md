## 1) Armington 替代弹性 & 需求 / 供给弹性

### 1.1 Armington 弹性（进口来源 / 国别替代）

**1）文献范围（总体 / 制造业）**

* 全行业的 Armington 弹性元分析（Bajzik et al., 2020）收集了 3,524 个估计值，给出的“优选估计”中位数约 **3.8**，推荐区间 **2.5–5.1**，制造业和中间品普遍高于农产品和服务。
* 这可以作为你模型里**“世界贸易一般水平”**的背景：σ 越大，贸易量对相对价差越敏感。

**2）美国 ITC CGE 模型中与半导体最接近的部门**

USITC 的《Revised Armington Elasticities of Substitution for the USITC Model》给出了 3 位 NAICS 部门的 Armington 弹性（国内 vs 进口）：

* **“Semiconductors and related devices”（半导体及相关器件，NAICS 334413）**

  * Armington 弹性：**1.7**
* **“Other electronic components”（其他电子元件）**

  * 弹性：**3.6**
* **“Electronic computers”**：1.0
* **“Communications equipment”**：2.2

> 这些是**国内 vs 总进口**的替代弹性（Armington），不是“进口来源之间”的二级弹性，但数量级可直接借用。

**3）模型建议（US / CN / ROW 三地区，高/中/低端）**

综合元分析（2.5–5.1）和 USITC 半导体部门（1.7），一个常用、保守但有文献支持的设置可以是：

* **Armington：国内 vs 进口（σD,M）**

  * 高端芯片：**1.5–2.0**，建议取 **1.7**（对标 USITC 半导体部门）。
  * 中端：**2.0**，略高一点体现更多替代。
  * 低端：**2.0–2.5**，建议取 **2.2**。
* **Armington：不同进口来源之间（σM,M’）**

  * 由于多国来源间可替代性通常大于“国内 vs 外国”，可以设为

    * 高端：**2.5–3.5**（建议 **3.0**）
    * 中端：**3–4**（建议 **3.5**）
    * 低端：**3–5**（建议 **4.0**）
  * 这让你的模型弹性落在元分析推荐区间 2.5–5.1 内，且与 USITC 半导体弹性（相对偏低）相容。

> 若你使用类似 GTAP 的两层 Armington 结构，可将 **domestic vs composite import = 1.7–2**，**within-import = 3–4**。

### 1.2 最终需求价格弹性 & 供给弹性

严格意义上的“半导体终端消费需求弹性”文献很少，多数 CGE 模型用的是**制造业可贸易品的一般参数**。GTAP 行为参数文档说明制造业部门的消费价格弹性与 Armington 弹性同阶，一般在 **|1|–|2| 左右**，但具体数值只有购买 GTAP 数据库才能看到。

结合一般制造业/电子产品的估计和主流 CGE 校准（推断）：

* **最终需求（半导体嵌入的电子产品）价格弹性**

  * 成熟制造品多在 **−0.5 ~ −1.5** 之间；高竞争度、可替代的 ICT 产品可达 **−2**。
  * 建议标定（作为芯片“使用方”终端需求价格弹性）：

    * 高端（高性能算力 / 数据中心等）：**−0.8 ~ −1.2**，建议 **−1.0**（较刚性）。
    * 中端（PC / 通用电子）：**−1.0 ~ −1.5**，建议 **−1.2**。
    * 低端（简单消费电子）：**−1.2 ~ −2.0**，建议 **−1.5**。

* **供给弹性（行业供给对价格的长期响应）**

  * 制造业的长期供给弹性（资本可调整）多在 **1–5** 之间，较资本密集行业通常在 **1.5–3**。GTAP 标准模型中，制造业部门的**要素替代弹性**和**产出转化弹性**也多落在这个区间。
  * 半导体因扩产周期长，但长期看资本能大幅响应：

    * 高端（先进制程）长期供给弹性：**1–2**，建议 **1.5**。
    * 中端：**2–3**，建议 **2.5**。
    * 低端（成熟制程 / 封测）：**2.5–4**，建议 **3.0**。

这些弹性最好在你做小规模校准（比如用 USITC / GTAP 结构跑一个对照实验）时再进行**敏感性分析**（例如 ±50%）。

## 2) 高/中/低端市场占比分拆

你给出的两种分拆方式：

* 按 **HS6：854231（处理器/控制器） / 854232（存储器） / 854239（其他 IC）**
* 按 **产品类型：逻辑 / 存储 / 模拟(analog) / 离散(discrete) / 传感 / 光电**

公开统计里，**HS6 分项的全球占比数据很难直接免费拿到**（需要 UN Comtrade 或 WITS 自己跑，并汇总），但按产品类型的全球销售占比可以直接从 SIA/WSTS 获得。

### 2.1 全球市场按产品类型的销售占比（WSTS / SIA）

SIA Factbook 基于 WSTS 数据指出，近几年全球半导体销售主要集中在 **Logic、Memory、Analog 和 Micro/MPU 等几大类，这几类合计约占 75% 以上的销售额**。

第三方对 WSTS 的公开摘要（例如 SIA 2022/2023 Factbook 的图表摘要）给出类似结构：

> 典型年份（约 2021–2023）的全球销售结构大致为（占全球销售额，大致范围）：
>
> * **Logic（含 MCU 之外的通用逻辑）**：约 **25–30%**
> * **Memory（DRAM + NAND）**：约 **20–30%**
> * **Microcomponents（MPU+MCU）**：约 **15–20%**
> * **Analog**：约 **15–20%**
> * **Discrete + Optoelectronics + Sensors & Actuators**：合计约 **10–15%**

你要的高/中/低端可以在此基础上做一个**建模分组假设**：

> **建议映射（可写入模型文档的假设）**
>
> * **高端**：高性能 Logic + MPU + 高密度 Memory（DRAM/NAND）
> * **中端**：其余 Microcomponents（MCU）、大部分 Analog、部分 Sensors/Opto
> * **低端**：Discrete、低附加值 Analog & Legacy 逻辑等

在这个映射下，一个合理且贴合 WSTS 的**全球销售占比近似**是：

* **高端芯片**：约 **50–60%** 全球销售

  * （Logic ~25–30% + 高端 Memory ~15–20% + 高性能 CPU/GPU/MPU 部分）。
* **中端芯片**：约 **25–35%**

  * （其余 Memory + Analog + 中高端 MCU / Sensors 等的一部分）。
* **低端芯片**：约 **10–15%**

  * （Discrete + 低端模拟/驱动等）。

> 这些比例属于在 WSTS 产品结构基础上的**分类假设**，你如果有 WSTS 付费表，可以进一步精确拆分到 2-digit / 4-digit product，以支持更精细的高/中/低划分。

## 3) 基期价格 / 单价（用于把贸易额换成数量）

**现实问题**：
公开渠道几乎不直接给出 **HS 854231/854232 的全球平均单价（$/unit）**，通常只给总值和重量或数量，需要你从 **USITC DataWeb / UN Comtrade** 自己算“单位价值”。因此我分两块说：

### 3.1 可作为量级参考的 “典型 ASP”

1. **低 / 中端：MCU、简单逻辑、模拟芯片**

   * 一份 MCU 市场研究（HardwareBee，总结多家厂商）指出，**2019 年全球微控制器平均售价约为 0.64 美元/颗**。
   * 这和市场共识一致：大量 8-bit / 16-bit MCU、简单驱动/电源 IC 的价格在 **0.3–2 美元** 之间。

2. **存储器（DRAM / NAND）**

   * 各类现货/合约价数据（如 DRAMeXchange、集邦咨询）显示，近几年 4–8Gb DRAM 或几百 GB 级 NAND 的单颗价格通常是**几美元到十几美元**，具体视容量与工艺节点而定。

3. **高端处理器 / GPU / 高性能 ASIC**

   * 公开报道中，PC/手机用的应用处理器 SoC 的出货 ASP 通常在 **20–60 美元/颗**，服务器 CPU/GPU/AI 加速卡的单价则从 **几百到几千美元** 不等（但后者往往包括封装 / 板卡等整套模组，不完全是 HS854231）。这些数值多见于厂商财报与投行分析报告（比如 NVIDIA、AMD 的产品线 ASP 分析）。

> 对于**“每颗 IC 的代表性价格”**，你可以采用下面这种极简、易用的结构（明显是“标定值而非统计值”，但与现实数量级一致）：
>
> * 高端：**100 美元/颗**（范围 **50–200**）
> * 中端：**10 美元/颗**（范围 **5–30**）
> * 低端：**1 美元/颗**（范围 **0.2–3**）

### 3.2 利用 USITC / Comtrade 数据自算 HS854231/854232 单价

**数据来源**

* USITC DataWeb / UN Comtrade 提供：

  * Trade Value（美元）
  * Quantity（既可为个数，也可能是重量，如 kg）

**操作建议**

1. 对每个 HS6、每个贸易流（US import/export）、每年，计算：
   [
   p_{h} = \frac{\text{Trade Value}*{h}}{\text{Quantity}*{h}}
   ]

   * 若单位是“个 (No.)”，就是 **$/unit**；
   * 若单位是 kg，则你可以用一个行业假设，比如“每公斤约含 N 颗芯片”来把 $/kg 转为 $/chip，或者直接把数量定义为“物理重量”，在模型里使用“公斤”作为数量单位。
2. 用最近若干年的平均（例如 2018–2022）平滑掉周期波动。

> 由于你已经从 USITC 下载过 8542 相关 CSV，**这个方法可以得到真正的数据驱动的基期价格**，然后用上面给的 ASP 区间做 sanity check 即可。

## 5) R&D：强度、分部门差异、技术进步效率 φ

### 5.1 半导体行业 R&D 占销售比例（整体）

SIA Factbook 及其中文摘要明确指出：

* **美国半导体行业 R&D 支出长期保持在销售额的 15% 以上**，过去 20 年基本在 15–20% 区间，是所有美国主要行业中 R&D 强度最高的之一。
* 2023 年 SIA 的《State of U.S. Semiconductor Industry》进一步指出：根据欧盟 2024 R&D 投资记分板，美国半导体行业 2023 年 **R&D 占收入 19.5%**，仅次于制药/生物技术。

> **模型建议值（整体行业）**
>
> * U.S. 半导体：R&D / Sales = **18–20%**，建议 **19%**。
> * Global industry：R&D / Sales ≈ **15–17%**（美国略高于全球平均）。

### 5.2 高/中/低端的 R&D 强度差异（用公司类型 proxy）

虽然没有直接按“高/中/低端芯片”切分的统计，但可以用典型公司来 proxy：

1. **高端数字 / GPU / AI（高端）**

   * NVIDIA：近几年 R&D 占收入比在 **10–27%** 之间波动，2023 年约 **27%**，随后因收入暴涨降至约 10%（TTM 口径）。
   * 这类公司集中在高性能逻辑与 GPU/AI 芯片，代表**高端逻辑/算力芯片**的 R&D 强度。

2. **模拟 / 嵌入式（主要是中端 / 低端）**

   * 德州仪器（TI）2024 年财报中，全年 R&D 支出约 **19.59 亿美元**，收入约 **156.4 亿美元**，R&D 强度 ≈ **12.5%**，以 analog、embedded 为主。

> 综合来看，可以为你的**高/中/低端部门设定如下 R&D 强度**：
>
> * 高端（先进逻辑、GPU、先进存储）：**18–25%**，建议 **22%**。
> * 中端（模拟、MCU、车规、工业控制）：**10–15%**，建议 **12%**。
> * 低端（离散器件、legacy 节点）：**5–10%**，建议 **8%**。

这与 “行业平均 ≈ 15–20%” 的事实相容：高端略高，中低端略低。

### 5.3 技术进步效率 φ（R&D → 生产率）

你要的 φ 本质上是生产函数中 **TFP 对 R&D 资本存量的弹性**。直接给 “半导体 φ” 的文献很少，但有若干实证给出了**R&D 资本的回报率 / 弹性**，可以用作参考：

* 《Return to R&D Investment and Spillovers in the Chinese Semiconductor Industry》（MIT 工作论文）用 1998–2005 年中国半导体企业面板数据估计了一个 Cobb-Douglas 生产函数：

  * 对于有 R&D 的半导体企业，**自有 R&D 资本的回报率约 5%**（整体），
  * 全球导向 segment（出口占比高）的回报率约 **15%**，
  * 国内导向 segment 回报率约 **3%**。
  * 这可以理解为：产出对“R&D 资本存量”的弹性大致在 **0.03–0.15** 的区间。

> **模型建议（作为 φ 的量级）**
>
> * 对高端芯片部门：
>
>   * φ_H ≈ **0.10–0.15**（推荐 **0.12**），对应“每长期 1% 的 R&D 资本增加带来约 0.12% 的产出增量”。
> * 对中端部门：
>
>   * φ_M ≈ **0.05–0.10**（推荐 **0.08**）。
> * 对低端部门：
>
>   * φ_L ≈ **0.03–0.05**（推荐 **0.04**）。

如果你用的是 “技术进步率 = φ·(R&D/Y)” 的形式，而不是对 **R&D 资本** 的弹性，可以把上述 φ 与长期稳态增长率对齐（例如设 TFP 增长 3–4%/年、R&D 强度 15–20%，则 φ 大概在 0.15–0.25 之间），这个转换可以在你的模型技术说明里写清楚。

## 6) GTAP / OECD ICIO 中电子行业的参数

公开文档中很少直接列出 **具体行业=电子设备/计算机通信设备的 Armington 与供需弹性数值表**，因为这些通常属于 GTAP 数据产品的一部分，只对付费用户开放。

但可以从 GTAP 行为参数文档中得出几个对你建模有用的结论：

1. **Armington 结构**

   * GTAP 采用两层 Armington：国内 vs 进口（ESUBD），以及不同进口来源之间（ESUBM）。
   * 制造业（包括电子设备）的 ESUBD 和 ESUBM 一般设在 **2–6** 区间，中间品和可贸易制成品的弹性更高，服务业更低。
   * 某些针对贸易战的应用中，研究者会做“**贸易弹性 ×2**”的敏感性实验，说明基准值并不算极高。

2. **最终需求弹性**

   * GTAP 使用 CDE 或 LES 形式标定私人消费，对制造业产品的自价格弹性通常在 **−0.5 ~ −1.5**，与你在 1.2 中采用的数值相容。

3. **OECD ICIO**

   * OECD ICIO 本身是投入产出表，并不附带 Armington/需求弹性；但很多基于 ICIO 的 CGE/贸易模型会引用 GTAP 或元分析的参数。

> 因此，从 GTAP/OECD 体系出发，你可以把本文第 1 部分给出的 Armington / 需求弹性理解为：
>
> * **数值区间与 GTAP 中制造业部门一致（2–4 级别）**，但针对半导体的高技术特性做了略保守的设定（特别是国内 vs 进口的弹性）；
> * 在没有更细微数据的情况下，这是符合主流 CGE 实践的选择。
